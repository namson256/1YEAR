# 🎓 1 Year | Foundations of Computer Science & Programming

Welcome to the definitive repository for **Year 1**. This space serves as a central hub for all foundational concepts in discrete logic, core computational thinking, systems analysis architectures, and basic statistical data processing across Semester 1 and Semester 2.

---

## 📅 Semester 1

### 🔢 Discrete Structure
> **`COURSE OVERVIEW`**
> Introduces the mathematical structures that form the backbone of computer science. Key topics include set theory, logic gates, propositional calculus, functions, relations, graph theory, combinatorics, and proof techniques critical for structural problem-solving.

* **💡 Key Reflection:** This was my first true introduction to thinking like a computer scientist. Moving away from standard algebra and into logic gates and set proofs was tough initially, but it taught me how to break complex computational scenarios down to their absolute core truths—a mindset that has massively upgraded my approach to writing logic.

---

### 💻 Programming Technique I
> **`COURSE OVERVIEW`**
> Covers fundamental concepts of structured and procedural programming. Students explore core algorithmic construction using pseudocode and flowcharts, alongside hands-on language execution covering data types, control structures (loops/conditionals), functions, and basic single-dimensional arrays.

* **💡 Key Reflection:** This module was where my software engineering journey truly started. Writing my first functioning loop and understanding how data flows down a program script was incredibly exciting. It grounded me in clean coding principles and taught me patience when reading terminal compilation errors.

---

### 🌐 Technology and Information System
> **`COURSE OVERVIEW`**
> Provides a macro-level overview of modern computing environments and evolving digital ecosystems. Topics explore hardware architectures, networks, security frameworks, cloud systems, and emerging technologies like Big Data and Artificial Intelligence.

* **💡 Key Reflection:** This module gave me an invaluable bird's-eye view of the fast-paced data landscape. It allowed me to see exactly how my computer science studies connect with real-world industries, solidifying why choosing a pathway like Data Engineering is so relevant in the modern tech market.

---

### 🎛️ Digital Logic
> **`COURSE OVERVIEW`**
> Deep dives into the fundamental building blocks of digital electronic systems. Students learn about binary number systems, Boolean algebra reduction, Karnaugh maps (K-maps), and the design of combinational and sequential logic circuits such as adders, multiplexers, and flip-flops.

* **💡 Key Reflection:** Digital Logic pulled back the outer curtain to reveal how hardware translates math into electricity. Simplifying complex Boolean equations using K-maps felt like playing a puzzle game, and mapping how physical circuits execute logical commands gave me a deep appreciation for the physical architecture running our code.

## 📅 Semester 2

### 📈 Probability & Statistical Data Analysis
> **`COURSE OVERVIEW`**
> Focuses on methods for collecting, organizing, analyzing, and presenting data to draw meaningful mathematical conclusions. Key areas include descriptive statistics, probability distributions, sampling behaviors, hypothesis testing, and foundational linear regression models.

* **💡 Key Reflection:** As an aspiring Data Engineer, this course felt immensely foundational. It proved to me that data processing isn't just about code, it's heavily rooted in mathematical validation. Figuring out variance distributions and running predictive regressions shifted my focus entirely toward analytical accuracy.

---

### 📦 Programming Technique II
> **`COURSE OVERVIEW`**
> Bridges procedural programming with complex application construction. This module introduces multi-dimensional arrays, advanced file stream manipulation, pointer tracking, memory allocation mechanics, and an introductory look into structural object blueprints.

* **💡 Key Reflection:** PT2 significantly pushed my capabilities beyond basic scripting. Handling file pathways and pointers forced me to think deeply about how applications manipulate disk resources and active machine memory. It served as a perfect stepping stone into object-oriented concepts.

---

### 🏢 System Analysis and Design `WBL`
> **`COURSE OVERVIEW`**
> Introduces the systematic processes of planning, analyzing, modeling, and designing software information systems. Through the Work-Based Learning framework, students learn how to gather client requirements, draft unified modeling language schemas, and plan complete software architectures.

* **💡 Key Reflection:** This class proved to me that writing functional code is completely useless if you build the wrong tool for the client. Stepping through object models and use-case analysis schemas showed me the immense engineering overhead required to orchestrate software systems correctly before a single line of script is ever generated.

---

### ⚙️ Computer Organization and Architecture
> **`COURSE OVERVIEW`**
> Analyzes the internal functional operations of modern computers. Topics cover instruction set architectures (ISA), register structures, basic assembly programming instructions, CPU execution pipelining, and multi-tier memory caching structures.

* **💡 Key Reflection:** Transitioning from high-level languages into low-level assembly syntax was a beautifully challenging experience. Pushing variables directly into registers and tracking clock cycles showed me the immense abstraction high-level code provides us, making me a much more resource-conscious engineer.

---

### 👥 Human Computer Interaction
> **`COURSE OVERVIEW`**
> Explores the methodologies, psychological frameworks, and design principles required to create intuitive, accessible, and user-centric digital layouts. Topics cover user research, wireframing, interactive prototyping, and empirical usability evaluation loops.

* **💡 Key Reflection:** HCI constantly reminded me that software is ultimately constructed for real, complex human beings. Building accessible interactive layouts taught me to drop my biases as a developer and evaluate interfaces purely through user accessibility, emphasizing that great engineering must always match functional usability.
